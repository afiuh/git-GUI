# 设计系统 – 灵感源自 Expo

## 1. 视觉主题与氛围

Expo 的界面是一种明亮且令人信赖的开发者平台，其核心理念是：用于构建应用的工具，其体验应该和构建出的应用本身一样精致。整个界面栖息在明亮、通透的画布上——一种偏冷的米白色（`#f0f0f3`）赋予了页面温和的技术感，避免纯白色的生硬。这是一个会呼吸的网站：章节之间巨大的垂直间距营造出画廊般的节奏，每个特性都有自己独立的“房间”。

设计语言高度克制——纯黑色（`#000000`）标题搭配尽可能浅的背景，而冷蓝灰色系（`#60646c`、`#b0b4ba`、`#555860`）负责所有的次要信息。界面本身几乎完全不含装饰性色彩；当色彩出现时，也只用于产品截图、应用图标和 React 宇宙插画——这样实际内容就在中性画布上显得生机勃勃。

Expo 的独特之处在于它的**药丸形几何**。按钮、标签页、视频容器甚至图片都使用大圆角或完全药丸形的圆角（24px–9999px），营造出一种有机、可亲的感受，与典型的尖锐开发者工具美学形成反差。配合超大标题上紧凑的字间距（64px 时 -1.6px 至 -3px），最终的设计既高级又友好——就像为开发者重新设计的 Apple 产品页面。

**关键特征：**
- 明亮的冷白画布（`#f0f0f3`），画廊般的垂直间距
- 严格单色：纯黑标题、冷蓝灰正文、无装饰性色彩
- 无处不在的药丸形几何——按钮、标签页、容器、图片（24px–9999px 圆角）
- 超大展示标题（64px）配合极端负字距（-1.6px 至 -3px）
- 唯一字体 Inter，使用 400–900 字重以发挥完整表现力
- 轻柔到几乎无法察觉的阴影，仅将元素微微抬离表面
- 产品截图是界面中唯一的色彩来源

## 2. 色彩板与角色

### 主色
- **Expo 黑**（`#000000`）：绝对的锚点——用于主标题、行动按钮和品牌标识。纯黑搭配冷白创造出最大对比但绝不刺眼。
- **近黑**（`#1c2024`）：正文的主要文字颜色——一种几乎察觉不到的蓝黑，相比纯黑更柔和，适合长时间阅读。

### 次要与强调色
- **链接钴蓝**（`#0d74ce`）：标准链接颜色——一种可信、饱和的蓝色，指示可交互性，且不破坏单色层次。
- **法律蓝**（`#476cff`）：用于页脚/法律链接的更亮饱和蓝色——比链接钴蓝稍抓眼球。
- **部件天蓝**（`#47c2ff`）：用于部件品牌元素的浅青色——系统中最亮的强调色。
- **预览紫**（`#8145b5`）：用于“预览”或 beta 特性指示的浓郁紫色，与标准内容形成清晰区分。

### 表面与背景
- **云灰**（`#f0f0f3`）：主要页面背景——一种带极轻微蓝紫调的冷米白。不暖、不生硬——精准的技术感。
- **纯白**（`#ffffff`）：卡片表面、按钮背景和提升的内容容器。在云灰之上形成清晰的“抬升”区别。
- **部件暗色**（`#1a1a1a`）：用于深色主题部件和覆盖元素的暗色表面。
- **横幅暗色**（`#171717`）：最暗的表面变体，用于推广横幅和高对比容器。

### 中性色与文本
- **石板灰**（`#60646c`）：主力辅助文本颜色（出现 305 次）。一种权威却不沉重的冷蓝灰色。
- **中石板**（`#555860`）：比石板灰略深，用于需要强调的辅助文本。
- **银色**（`#b0b4ba`）：三级文本、占位符和降低强调的元数据。可读舒适但明显退居后台。
- **锡器色**（`#999999`）：深色背景中的折叠图标和极度降级的 UI 元素。
- **浅银**（`#cccccc`）：深色背景中的箭头图标和装饰元素。
- **暗石板**（`#363a3f`）：深色表面边框、开关轨道和强调的容器。
- **炭色**（`#333333`）：暗色模式开关背景和深层次的次级表面。

### 语义与强调色
- **警告琥珀**（`#ab6400`）：温暖的深琥珀色，用于警告状态——特意不采用亮黄色，传达严肃性。
- **破坏玫瑰**（`#eb8e90`）：柔和的粉珊瑚色，用于禁用状态的破坏性操作——比通常的红色更温和，减少警报疲劳。
- **边框薰衣草**（`#e0e1e6`）：标准卡片/容器边框——一种可见但不厚重的冷薰衣草灰色。
- **输入框边框**（`#d9d9e0`）：按钮和表单元素边框——相比卡片边框稍暖/稍深，以区分交互元素。
- **暗色焦点环**（`#2547d0`）：深色主题中键盘焦点指示器的深蓝色。

### 渐变系统
- 值得注意的是，该设计在**界面层完全没有渐变**。视觉丰富度来自产品截图、React 宇宙插画以及精心的阴影层次，而非颜色渐变。这种“缺失”本身就是一种设计决策——渐变会破坏那种临床般的精准感。

## 3. 字体规则

### 字体族
- **主字体**：`Inter`，备用：`-apple-system, system-ui`
- **等宽字体**：`JetBrains Mono`，备用：`ui-monospace`
- **系统备用**：`system-ui, Segoe UI, Roboto, Helvetica, Arial, Apple Color Emoji, Segoe UI Emoji`

### 层级

| 角色 | 字体 | 大小 | 字重 | 行高 | 字间距 | 备注 |
|------|------|------|--------|-------------|----------------|-------|
| 展示 / 主标题 | Inter | 64px (4rem) | 700–900 | 1.10 (紧凑) | -1.6px 至 -3px | 最大冲击力，极致紧缩 |
| 章节标题 | Inter | 48px (3rem) | 600 | 1.10 (紧凑) | -2px | 特性区域锚点 |
| 子标题 | Inter | 20px (1.25rem) | 600 | 1.20 (紧凑) | -0.25px | 卡片标题、特性名称 |
| 大正文 | Inter | 18px (1.13rem) | 400–500 | 1.40 | normal | 介绍段落、章节描述 |
| 正文 / 按钮 | Inter | 16px (1rem) | 400–700 | 1.25–1.40 | normal | 标准文本、导航链接、按钮 |
| 说明 / 标签 | Inter | 14px (0.88rem) | 400–600 | 1.00–1.40 | normal | 描述、元数据、徽章文本 |
| 标签 / 小号 | Inter | 12px (0.75rem) | 500 | 1.00–1.60 | normal | 最小的无衬线文本、徽章 |
| 代码正文 | JetBrains Mono | 16px (1rem) | 400–600 | 1.40 | normal | 行内代码、终端命令 |
| 代码说明 | JetBrains Mono | 14px (0.88rem) | 400–600 | 1.40 | normal | 代码片段、技术标签 |
| 代码小号 | JetBrains Mono | 12px (0.75rem) | 400 | 1.60 | normal | 大写的技术标签 |

### 原则
- **单一字体，充分表达**：Inter 是唯一的无衬线字体，字重从 400（常规）到 900（黑色）。这让设计有了统一的声音，同时在极轻的正文和雷鸣般的展示标题之间实现戏剧性对比。
- **大字号极端负字距**：64px 标题使用 -1.6px 到 -3px 字间距，形成超密集的文本块，如同标志字体。这种激进压缩是标志性的排版手法。
- **字重即层次**：700–900 用于展示，600 用于标题，500 用于强调，400 用于正文。层次分明，没有模糊的中间字重。
- **一致的 1.40 正文行高**：几乎所有正文和 UI 文本共享 1.40 行高，形成有节奏的垂直一致性。

## 4. 组件样式

### 按钮

**主要（白底有边框）**
- 背景：纯白（`#ffffff`）
- 文本：近黑（`#1c2024`）
- 内边距：0px 12px（紧凑，高度由内容驱动）
- 边框：细实线 输入边框（`1px solid #d9d9e0`）
- 圆角：适度圆角（6px）
- 阴影：悬停时有微妙的组合阴影
- 低调的默认状态——干净、专业、不张扬

**主要药丸形**
- 与主要按钮相同，但圆角为药丸形（9999px）
- 用于主区域行动按钮和高强调操作
- 额外的圆润度暗示“从这里开始”

**暗色主要**
- 背景：Expo 黑（`#000000`）
- 文本：纯白（`#ffffff`）
- 药丸形（9999px）或大圆角（32–36px）
- 无边框（黑色本身就是边框）
- 最高强调度的行动按钮——仅用于主要转化操作

### 卡片与容器
- 背景：纯白（`#ffffff`）——在云灰背景上明显抬升
- 边框：细实线 边框薰衣草（`1px solid #e0e1e6`）用于标准卡片
- 圆角：标准卡片舒适圆角（8px）；特色容器大圆角（16–24px）
- 阴影等级 1：轻语（`rgba(0,0,0,0.08) 0px 3px 6px, rgba(0,0,0,0.07) 0px 2px 4px`）——几乎察觉不到的抬升
- 阴影等级 2：标准（`rgba(0,0,0,0.1) 0px 10px 20px, rgba(0,0,0,0.05) 0px 3px 6px`）——清晰的浮动感
- 悬停：可能加深阴影或改变背景

### 输入框与表单
- 背景：纯白（`#ffffff`）
- 文本：近黑（`#1c2024`）
- 边框：细实线 输入边框（`1px solid #d9d9e0`）
- 内边距：0px 12px（与按钮保持一致）
- 圆角：适度圆角（6px）
- 聚焦：蓝色环阴影（通过 CSS 自定义属性）

### 导航
- 顶部固定导航，背景透明/模糊
- Logo：黑色 Expo 文字标识
- 链接：近黑（`#1c2024`）或石板灰（`#60646c`），14–16px Inter 字重 500
- 行动按钮：右侧黑色药丸按钮（“Sign Up”）
- GitHub star 徽章作为社交证明
- 状态指示器（“All Systems Operational”）带绿点

### 图像处理
- 产品截图和设备模型是视觉主角
- 视频和图像容器使用大圆角（24px）
- 截屏显示在逼真的设备框架内
- 深色 UI 截屏与浅色画布形成对比
- 在圆角容器内全幅展示

### 特色组件

**Universe React Logo**
- 动画/插画形式的 React 标识作为视觉中心
- 将 Expo 身份与 React 生态连接起来
- 页面上唯一的插画元素

**设备预览网格**
- 同时展示多种设备类型（手机、平板、Web）
- 直观展示跨平台能力
- 每个设备都带有逼真的设备外框

**状态徽章**
- 导航栏中的“All Systems Operational”药丸
- 绿点 + 文本——紧凑的信任信号
- 药丸形（36px 圆角）

## 5. 布局原则

### 间距系统
- 基本单位：8px
- 标尺：1px, 2px, 4px, 8px, 12px, 16px, 24px, 32px, 40px, 48px, 64px, 80px, 96px, 144px
- 按钮内边距：0px 12px（异常紧凑 —— 高度由行高决定）
- 卡片内部内边距：约 24–32px
- 章节垂直间距：极大（估计主要章节之间 96–144px）
- 组件间隙：16–24px 兄弟元素之间

### 网格与容器
- 最大容器宽度：约 1200–1400px，居中
- 主区域：单列居中，呼吸空间巨大
- 特性章节：交替布局（左图右文、右图左文、全宽展示）
- 卡片网格：2–3 列用于特性亮点
- 全宽章节内部包含居中的内容

### 空白哲学
- **画廊般的节奏**：每个章节都像独立的展览，被大量空白包围。这营造出高级、不匆忙的浏览体验。
- **留白即设计**：慷慨的空白本身就是主要设计元素——它传递自信、品质，以及每个特性都值得单独关注的信息。
- **内容孤岛**：章节如孤岛漂浮在空白中，通过滚动连接，而非视觉上的连续。

### 圆角标尺
- 近似直角（4px）：小型内联元素、标签
- 适度圆角（6px）：按钮、表单输入、组合框 —— 功能交互圆角
- 舒适圆角（8px）：标准内容卡片、容器
- 大圆角（16px）：特性标签页、内容面板
- 很大圆角（24px）：按钮、视频/图像容器、标签面板 —— 标志性软圆角
- 超大圆角（32–36px）：主区域行动按钮、状态徽章、导航按钮
- 药丸形（9999px）：主要行动按钮、标签、头像 —— 最大友好度

## 6. 深度与投影

| 等级 | 处理方式 | 用途 |
|-------|-----------|------|
| 平面（等级 0） | 无阴影 | 云灰页面背景、内联文本 |
| 表面（等级 1） | 白背景，无阴影 | 云灰背景上的标准白卡片 |
| 轻语（等级 2） | `rgba(0,0,0,0.08) 0px 3px 6px` + `rgba(0,0,0,0.07) 0px 2px 4px` | 轻微卡片抬升、悬停状态 |
| 抬高（等级 3） | `rgba(0,0,0,0.1) 0px 10px 20px` + `rgba(0,0,0,0.05) 0px 3px 6px` | 特性展示、产品截图 |
| 模态（等级 4） | 深色遮罩（`--dialog-overlay-background-color`）+ 重阴影 | 对话框、覆盖层 |

**阴影哲学**：Expo 将阴影用作轻柔的低语，而非建筑式的宣言。主要的深度机制是 **背景色对比** —— 白卡片浮在云灰之上，而非通过投影。当阴影出现时，它们是柔和、弥散且方向向下（投影）的，营造出纸片悬在桌面几毫米上的感觉。

## 7. 做与不做

### 做
- 使用云灰（`#f0f0f3`）作为页面背景，纯白（`#ffffff`）作为抬升卡片 —— 这种双色浅色系统是核心
- 展示标题使用极端的负字距（64px 时 -1.6px 到 -3px），获得标志性的紧缩感
- 主要行动按钮使用药丸形圆角（9999px）—— 有机形状是身份核心
- 为标题和主要行动按钮保留黑色（`#000000`）—— 它在浅色画布上承载最大权威
- 使用石板灰（`#60646c`）作为辅助文本 —— 它在可读和退后之间取得了精确平衡
- 保持章节之间巨大的垂直间距（96px+）—— 画廊般的节奏定义了高级感
- 使用产品截图作为主要视觉内容 —— 界面保持单色，产品带来色彩
- 使用 Inter 全字重范围（400–900）—— 字重对比就是层次

### 不做
- 不要在界面镀铬中引入装饰性色彩 —— 单色板是刻意的
- 不要在交互元素上使用尖锐角（border-radius < 6px）—— 药丸/圆润几何是标志
- 不要将章节间距减少到 64px 以下 —— 呼吸空间就是设计
- 不要使用沉重的投影 —— 深度来自背景对比和极轻阴影
- 不要混入额外字体 —— Inter 处理从展示到说明的所有角色
- 不要在正文中使用比 -0.25px 更宽的字间距 —— 极端紧缩仅保留给展示
- 不要使用比 2px 更宽的边框 —— 边框是克制的，通过背景色和柔和边界实现
- 不要给界面添加渐变 —— 视觉丰富度来自内容，而非装饰
- 不要在语义上下文之外使用饱和颜色 —— 调色板严格是灰度 + 功能性蓝色

## 8. 响应式行为

### 断点
| 名称 | 宽度 | 主要变化 |
|------|-------|-------------|
| 移动端 | <640px | 单列、汉堡菜单、卡片堆叠、主标题缩至约 36px |
| 平板 | 640–1024px | 2 列网格、导航紧凑、中等主标题 |
| 桌面 | >1024px | 完整多列布局、展开导航、超大主标题（64px） |

*只检测到一个明确的断点（640px），暗示这是一个基于流体容器查询或 `min()`/`clamp()` 的响应式系统，而非固定断点。*

### 触摸目标
- 按钮使用大圆角（24–36px），形成大而适合手指的表面
- 导航链接间距充足
- 状态徽章尺寸适合触摸（36px 圆角）
- 最小推荐：44x44px

### 折叠策略
- **导航**：全横向导航带行动按钮在移动端折叠为汉堡菜单
- **特性章节**：多列 → 堆叠单列
- **主标题文本**：64px → 约 36px 渐进缩放
- **设备预览**：网格 → 堆叠/轮播
- **卡片**：并排 → 垂直堆叠
- **间距**：按比例减少，但保持慷慨的节奏

### 图像行为
- 产品截图按比例缩放
- 设备模型可能在移动端简化或显示更少的设备
- 在所有尺寸下保持圆角
- 为折叠下方内容使用懒加载

- “构建一个导航栏：左侧 Expo logo，文本链接使用近黑 (#1c2024) 14px Inter 字重 500，右侧黑色药丸行动按钮。背景透明并带模糊背景。底部边框 1px 边框薰衣草 (#e0e1e6)。”
- “创建一个代码块，使用 JetBrains Mono 14px，纯白表面，边框薰衣草，8px 圆角。代码用近黑，关键词用链接钴蓝 (#0d74ce)。”
- “设计一个状态徽章药丸（9999px 圆角），带绿点和‘All Systems Operational’文本，12px Inter 字重 500。背景纯白，边框 1px 输入边框 (#d9d9e0)。”

### 迭代指南
1. 一次聚焦一个组件
2. 引用具体的颜色名称和十六进制码 —— “使用石板灰 (#60646c)” 而非 “把它变灰”
3. 有目的地使用圆角值 —— 按钮用 6px，卡片用 8px，图片用 24px，药丸用 9999px
4. 在测量值旁边描述“感觉” —— “96px 章节间距，巨大的呼吸空间”
5. 始终指定 Inter 和确切字重 —— 字重对比就是层次
6. 对于阴影，从投影表中指定“轻语阴影”或“标准抬高”
7. 保持界面单色 —— 让产品内容成为色彩来源
# Git GUI 设计指南 – 基于 Expo 设计系统

本指南继承 Expo 设计系统的所有优秀规则（明亮画布、单色板、圆润几何、字体层级、轻量阴影等），并根据 **Git 图形化客户端** 的特殊需求进行扩展。

## 适用场景
- 一个面向开发者的 Git 操作工具，功能包括：变更管理（文件状态）、快照历史、分支管理、远程同步。
- 需要同时处理大量列表项（文件、提交记录）、代码差异显示、合并冲突提示等。

## 一、主题与基调（继承 Expo，增加暗色主题）

### 默认浅色主题（完全遵循 Expo）
- 页面背景：云灰 `#f0f0f3`
- 卡片表面：纯白 `#ffffff`
- 主文字：近黑 `#1c2024`
- 辅助文字：石板灰 `#60646c`
- 边框：边框薰衣草 `#e0e1e6`
- 链接/焦点：链接钴蓝 `#0d74ce`

### 暗色主题（新增，供开发者选择）
- 页面背景：深空灰 `#121212`（或 `#1a1a1a`）
- 卡片表面：暗炭色 `#2c2c2c`
- 主文字：浅灰 `#e5e5e5`
- 辅助文字：银灰 `#a0a0a0`
- 边框：暗石板 `#363a3f`
- 链接/焦点：亮钴蓝 `#3b82f6`
- **切换方式**：顶部栏提供浅色/暗色图标切换，保存用户偏好。

## 二、Git 专属颜色语义（在保持单色基调的前提下增加）

这些颜色仅用于 **区分文件状态**，不破坏整体设计风格。

| Git 状态 | 浅色主题颜色 | 暗色主题颜色 | 使用位置 |
| :--- | :--- | :--- | :--- |
| **未暂存的改动**（红色） | `#e5484d`（红，饱和度适中） | `#f87171`（亮红） | 文件列表中的文件名、状态徽章 |
| **已暂存的改动**（绿色） | `#2e7d32`（绿，偏深） | `#4ade80`（亮绿） | 文件列表中的文件名、状态徽章 |
| **未跟踪的文件**（灰色） | `#6c6f78`（中灰） | `#9ca3af`（浅灰） | 文件列表中的文件名 |
| **冲突文件**（警告色） | 使用 Expo 的警告琥珀 `#ab6400` 作为背景条或边框 | `#f59e0b` | 合并冲突时，文件行高亮 |
| **Diff 添加行** | 浅绿背景 `#e6f4ea`，文字 `#1e4620` | 深绿背景 `#1a3b2c`，文字 `#bbf7d0` | diff 视图中的 `+` 行 |
| **Diff 删除行** | 浅红背景 `#fce8e8`，文字 `#8b3a3a` | 深红背景 `#3b2626`，文字 `#fecaca` | diff 视图中的 `-` 行 |
| **Diff 行号** | `#b0b4ba`（银灰） | `#6b7280` | 差异视图左侧行号列 |

> **规则**：这些颜色仅在必要的地方使用（文件状态标记、diff 高亮），**不得**用于按钮、标题或装饰性元素，以维持 Expo 的单色权威感。

## 三、布局：针对 Git GUI 的三栏结构

沿用 Expo 的卡片式、大间距原则，设计如下布局：

+--------------------------+------------------------------+-----------------------------+
|      左侧栏              |         中间栏                |         右侧栏               |
|  （分支管理）            |  （变更管理/文件状态）        |  （快照管理/历史+diff）      |
|                          |                              |                             |
| - 当前分支（高亮）       | - 未暂存改动（红色）          | - 提交列表（短哈希+说明）     |
| - 本地分支列表           | - 已暂存改动（绿色）          | - 点击提交显示：             |
| - 操作按钮：             | - 未跟踪文件（灰色）          |   · 改动文件列表             |
|   · 新建分支             |   · 每个文件带操作按钮：       |   · 文件 diff（双栏）        |
|   · 切换（双击）         |       · 暂存 / 取消暂存        |                             |
|   · 合并到当前           |       · 丢弃修改              |                             |
|   · 删除                 | - 底部：提交信息输入框 + 提交按钮 |                             |
+--------------------------+------------------------------+-----------------------------+
|                        顶部栏（仓库路径、远程状态、刷新、主题切换）                       |
+----------------------------------------------------------------------------------------+
### 尺寸与间距（继承 Expo）

- 三栏之间使用 `24px` 间距
- 每个卡片内部内边距 `24px`
- 文件列表每行高度 `48px`（确保触摸目标）
- 提交历史每项高度 `56px`（包含较多元数据）
- 所有卡片圆角 `8px`，主要按钮圆角 `9999px`

## 四、Git 特有组件样式

### 1. 文件列表项

css

```
.file-item {
  display: flex;
  align-items: center;
  justify-content: space-between;
  padding: 12px 16px;
  border-bottom: 1px solid var(--border-color);
  background: var(--card-bg);
}
.file-item.status-modified { color: var(--git-red); }
.file-item.status-staged  { color: var(--git-green); }
.file-item.status-untracked { color: var(--git-gray); }
.file-item.status-conflict { background: var(--git-warning-bg); border-left: 3px solid var(--warning-amber); }
```

- 每个文件旁放置三个图标按钮（暂存/取消暂存、丢弃），使用 Expo 药丸形按钮风格（背景透明，边框，悬停变浅）。

### 2. Diff 视图（代码差异）

- 使用等宽字体 JetBrains Mono，14px。
- 双栏布局：左侧行号（浅色主题 `#b0b4ba`，暗色主题 `#6b7280`），右侧代码行。
- 每行显示 `+` / `-` 符号，背景色分别使用添加/删除色。
- 顶部显示文件路径，路径使用 Inter 14px 500，颜色石板灰。
- 提供一个“复制完整 diff”按钮（药丸形小按钮）。

### 3. 提交历史列表

- 每行包括：短哈希（7位，使用等宽字体）、提交说明（Inter 16px 500）、作者、相对时间（如“2 hours ago”，使用石板灰 12px）。
- 当前分支的最新提交高亮（左侧蓝色竖条或淡淡背景）。
- 支持按说明关键词搜索（前端过滤），搜索框样式使用 Expo 输入框。

### 4. 合并冲突提示

- 当 `git merge` 产生冲突时，在中间栏顶部显示警告卡片： - 背景：浅琥珀色（浅色主题 `#fef5e7`，暗色主题 `#332a1a`） - 边框：警告琥珀 `#ab6400` - 文字：`conflict.md` 等冲突文件列表，每条带“使用我的版本”、“使用他们的版本”、“手动编辑”按钮（药丸形小按钮）。
- 手动编辑后，提供一个“标记为已解决并继续”按钮。

### 5. 分支切换保护

- 当用户尝试切换分支且有未提交变动时，弹出 Expos 风格的 Dialog（模态框，等级 4 阴影），提示“请先提交或暂存改动”，提供两个操作按钮：“暂存并切换”、“强制切换（丢弃改动）”（用破坏玫瑰色）。

## 五、交互与反馈（继承 Expo 并细化）

- **所有异步操作**（提交、推送、拉取、合并等）显示加载进度条（细线，位于顶部栏下方），同时禁用相关按钮，防止重复点击。
- **成功反馈**：右下角弹出 Expo 风格的轻提示（纯白背景，绿色边框，3 秒自动消失）。
- **错误反馈**：红色边框弹窗，显示原始 Git 错误信息（可折叠详情），并提供“复制错误”按钮。
- **空状态**：当没有改动时，中间栏显示一个友好的说明插画（React 宇宙风格的小图标）+ “工作区干净，没有什么需要提交的”文字。

## 六、响应式适配（针对 Git GUI 的特殊性）

- **宽屏（\>1400px）**：三栏可以按 1.2 : 2 : 1.5 比例分配宽度。
- **平板（640px–1024px）**：右侧栏（快照）可以折叠为抽屉（点击提交时从右侧滑入），保持中间栏始终可见。
- **移动端（<640px）**：默认只显示中间栏（变更管理），左侧栏和右侧栏通过底部浮动按钮触发（类似于 “Branches” 和 “History” 按钮）。

## 八、做与不做（Git GUI 专用补充）

### 做

- 文件状态颜色（红/绿/灰）仅在文件名字或状态徽章上使用，不要用于其他 UI 元素。
- 为 diff 视图提供浅色/暗色两种语法高亮背景，但背景饱和度要低，确保长时间阅读不刺眼。
- 在合并冲突时明确列出冲突文件，并提供便捷的按钮帮助用户快速选择版本。
- 每次切换分支或合并后，自动刷新文件列表和历史视图。

### 不做

- 不要使用饱和度超过 60% 的红/绿作为大面积背景（比如不要把整行都染红），仅用于文字或小徽章。
- 不要在正常状态下给文件行添加任何背景色（保持纯白/暗色卡片背景），只通过文字颜色区分状态。
- 不要添加复杂的动画（如弹跳、旋转），保持 Expo 式的克制过渡（0.2s 淡变）。
- 不要在主界面显示 Git 命令的输出日志，可以折叠到“开发者面板”中。

## 九、组件示例（基于 React）

jsx

```
// 文件列表项组件示例（遵循设计令牌）
<FileItem className={classnames({
  'status-modified': file.status === 'modified',
  'status-staged': file.status === 'staged',
  'status-untracked': file.status === 'untracked',
})}>
  <FileName>{file.path}</FileName>
  <ButtonGroup>
    {file.status === 'staged' ? (
      <PillButton onClick={() => unstage(file)}>取消暂存</PillButton>
    ) : (
      <PillButton onClick={() => stage(file)}>暂存</PillButton>
    )}
    <IconButton icon="trash" onClick={() => discard(file)} />
  </ButtonGroup>
</FileItem>
```
